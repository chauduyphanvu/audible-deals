# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in audible-deals, please report it responsibly.

**Do not open a public issue.** Instead, use [GitHub's private vulnerability reporting](https://github.com/chauduyphanvu/audible-deals/security/advisories/new) to submit your report.

Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgement:** within 48 hours
- **Initial assessment:** within 1 week
- **Fix or mitigation:** as quickly as possible, depending on severity

## Scope

This policy covers the `audible-deals` CLI tool and its published packages. It does not cover the upstream [audible](https://github.com/mkb79/Audible) library — please report issues with that project to its maintainers directly.

## Supported Versions

Only the latest release is actively supported with security fixes.

| Version | Supported |
|---------|-----------|
| Latest  | Yes       |
| Older   | No        |
